# BUG-020: Возможность оформления заказа с невалидным номером телефона

Окружение:
- Браузер: Chrome 145.0.7632.109
- ОС: Windows 11
- Устройство: PC
- Сайт: https://detalika.ru/

Предусловия:
- Пользователь находится на сайте
- Товар доступен для покупки

Шаги воспроизведения:
1. Перейти на сайт https://detalika.ru/
2. В навигационной панели нажать “Общий каталог”
3. Выбрать любую марку автомобиля (например, Audi)
4. Нажать кнопку “Купить” на любом товаре
5. Нажать кнопку “Добавить”
6. Перейти в корзину
7. Ввести имя
8. Ввести номер телефона: +71234567891
9. Указать улицу в поле “Способ получения”
10. Нажать кнопку “Заказать”

Фактический результат:
- Форма принимает невалидный номер телефона
- Заказ успешно оформляется

Ожидаемый результат:
- Форма должна валидировать номер телефона
- При вводе некорректного номера должно отображаться сообщение об ошибке
- Заказ не должен оформляться с невалидными данными

Серьезность: Значительная (Data Validation)  
Приоритет: Высокий  
Воспроизводимость: 5/5

Примечание:
Поле телефона проходит валидацию (подсвечивается зелёным), несмотря на некорректный формат
