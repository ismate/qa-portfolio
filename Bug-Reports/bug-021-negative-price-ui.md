# BUG-021: Некорректное отображение отрицательной цены при получении невалидных данных от сервера

Окружение:
- Браузер: Chrome 146
- ОС: Windows 11
- Устройство: PC
- Сайт: https://euroauto.ru

Предусловия:
- Пользователь находится в корзине с добавленным товаром
- Используется Charles Proxy (перехват и подмена response)

Шаги воспроизведения:
1. Открыть сайт https://euroauto.ru
2. Добавить товар в корзину
3. Перейти в корзину
4. Перехватить запрос получения данных товара (GET/POST)
5. Подменить в response значение цены:
   "price": -10000
6. Нажать кнопку увеличения количества товара

Фактический результат:
- Цена товара отображается как отрицательная (-10000)
- При увеличении количества сумма пересчитывается в отрицательную сторону (-20000, -30000)
- UI не обрабатывает некорректные значения

Ожидаемый результат:
- Интерфейс должен валидировать входящие данные
- Отрицательные значения цены не должны отображаться
- Должна быть защита от некорректных значений (fallback или ошибка)

Серьезность: Незначительная (UI / Data Handling)  
Приоритет: Средний  
Воспроизводимость: Всегда  

Примечания:
- Некорректные данные были получены искусственно (через подмену response)
- Баг демонстрирует отсутствие защиты UI от невалидных данных
